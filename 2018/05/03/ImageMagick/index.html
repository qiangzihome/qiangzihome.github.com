ImageMagick

发表于 2018-05-03

ImageMagick安装

yum install -y libpng-devel

yum install -y libjpeg-devel

./configure --enable-shared --disable-static --disable-openmp --prefix=/usr/local/ImageMagick

make && make install

sudo ldconfig /usr/local/lib

Jmagick 安装

yum install -y autoconf

yum install -y libtool       
yum install -y libtool-ltdl    
yum install -y libtool-ltdl-devel

find / -name config.sub

cp /usr/share/libtool/config/config.sub  /data/jmagick
cp /usr/share/libtool/config/config.guess .

./configure --prefix=/usr/local --with-magick-home=/usr/ImageMagick/include/ImageMagick-7 --with-java-home=/usr/java/jdk1.8.0_151

make clean
make all
make install

问题解决

1、System.out.println(System.getProperty(“java.library.path”)).
打印出当前环境的路径，然后再把 jmagick.dll或jmagick.so 复制到其中的一个路径文件夹中问题解决。

2、magick.MagickException: No image to scale
版本不匹配,要像下面这们,都是同一个版本号
ImageMagick-6.4.0-0.tar.gz
jmagick-6.4.0-src.tar.gz

3、magick.MagickException: Unable to retrieve handle
要调用 magick.MagickImage.destroyImages() 来关闭

4、no decode delegate for this image format
使用:identify -list format
判断有支持几种图片格式，没有的则需要安装对应的格式库

5、单线程转换每张图片大概50ms，两个线程却需要500ms
用convert –version 命令查看
机器不支持openMP导致需要重新编译 ./configure –disable-openmp 在进行安装
首先检查一下电脑里已经安装的ImageMagick版本：
convert -version

6、Exception in thread “main” java.lang.UnsatisfiedLinkError:
C:\WINDOWS\system32\jmagick.dll: Can’t find dependent libraries
解决：重启Eclipse。加载新的环境变量。
